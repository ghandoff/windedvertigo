# pocket-prompts → Cloudflare Workers migration

Playbook for moving `github.com/ghandoff/pocket-prompts` off Vercel onto a
Cloudflare Worker (`wv-pocket-prompts`) so the Vercel team can be deleted.

**Status:** Active. **Owner:** Garrett. **Deadline:** Before Vercel cancel.

Pocket-prompts is the last Vercel project keeping the team alive — once this
lands and the Vercel project is deleted, the team can be cancelled.

## Pre-flight (do once, on your laptop)

```sh
# 1. Clone (or cd into) the pocket-prompts repo
cd ~/code   # or wherever you keep repos
git clone https://github.com/ghandoff/pocket-prompts.git
cd pocket-prompts

# 2. Pull the helper script from this repo
cp ../windedvertigo/scripts/pocket-prompts-migrate.sh .
chmod +x pocket-prompts-migrate.sh

# 3. Make sure wrangler is authenticated to the right CF account
npx wrangler whoami
# expect: garrett@windedvertigo.com on account 097c92553b268f8360b74f625f6d980a
```

## Run the migration

```sh
./pocket-prompts-migrate.sh --dry-run    # preview every change
./pocket-prompts-migrate.sh              # actually do it
```

What the script does:

1. **Detects the framework**. Reads `package.json` for `next`, `nuxt`, `astro`, etc. Bails if unsupported — Next.js (the most likely case based on the project's prior Vercel runtime stats) is the only auto-handled framework.
2. **Strips `@vercel/*` runtime deps** and rewrites `process.env.VERCEL_URL` fallbacks to use a `WORKER_URL` env var (same approach we used in the monorepo).
3. **Adds CF tooling**: `@opennextjs/cloudflare`, `wrangler`, `@cloudflare/workers-types`.
4. **Drops in `wrangler.jsonc`** (template below) with `name: wv-pocket-prompts`.
5. **Drops in `open-next.config.ts`** and **`worker.ts`** (templates below).
6. **Adds npm scripts**: `build:cf`, `preview:cf`, `deploy:cf`.
7. **Adds `.gitignore` entries** for `.open-next/`.
8. **Commits everything** on a branch `claude/migrate-to-cloudflare-workers`.

The script does **NOT** push, **NOT** delete the Vercel project, **NOT** set
secrets. Those are your decisions. Continue with the manual steps below.

## After the script

```sh
# Set secrets — only those that actually exist on the Vercel project.
# Inspect your current Vercel env via dashboard or `vercel env ls` first.
npx wrangler secret put SOMETHING --name wv-pocket-prompts
# ...repeat for each secret your app uses

# First deploy (canary, workers.dev only)
npm run deploy:cf

# Verify
curl -sI https://wv-pocket-prompts.windedvertigo.workers.dev/ | head -10
# expect: HTTP/2 200 (or whatever the home route returns), cf-ray header,
# NO x-vercel-* headers.

# Open in browser, click through key flows. If it works:
git push origin claude/migrate-to-cloudflare-workers
# Open a PR, merge to main when happy.

# Custom domain (if pocket-prompts had one — Vercel API previously showed
# none, only *.vercel.app, so this step may be skipped):
# Add a CF Workers Route in the wrangler.jsonc routes block, then redeploy.

# Final: delete the Vercel project
# Vercel dashboard → pocket-prompts → Settings → bottom → Delete
```

## Templates (the script writes these; here for reference)

### `wrangler.jsonc`

```jsonc
{
  "$schema": "node_modules/wrangler/config-schema.json",
  "name": "wv-pocket-prompts",
  "main": "worker.ts",
  "account_id": "097c92553b268f8360b74f625f6d980a",
  "compatibility_date": "2025-04-01",
  "compatibility_flags": ["nodejs_compat"],
  "workers_dev": true,

  "assets": {
    "directory": ".open-next/assets",
    "binding": "ASSETS"
  },

  "services": [
    { "binding": "WORKER_SELF_REFERENCE", "service": "wv-pocket-prompts" }
  ],

  "vars": {
    "CF_WORKERS_ENV": "1"
  }

  // Secrets via: wrangler secret put <NAME> --name wv-pocket-prompts
  // Routes block: add only if you want a custom domain.
}
```

### `open-next.config.ts`

```ts
import { defineCloudflareConfig } from "@opennextjs/cloudflare";
export default defineCloudflareConfig({});
```

### `worker.ts`

```ts
// @ts-ignore — generated by `npx opennextjs-cloudflare build`; absent on fresh clone.
import openNextHandler, {
  // @ts-ignore
  DOQueueHandler,
  // @ts-ignore
  DOShardedTagCache,
  // @ts-ignore
  BucketCachePurge,
} from "./.open-next/worker.js";

export { DOQueueHandler, DOShardedTagCache, BucketCachePurge };

export default {
  fetch: openNextHandler.fetch,
};
```

Note: This worker.ts does **not** wrap with `@windedvertigo/security` because
pocket-prompts is a standalone repo without access to that internal package.
If you want CSP/HSTS headers, either:
(a) copy the relevant snippet from `packages/security/` in the windedvertigo
    monorepo into pocket-prompts as `lib/security.ts`, or
(b) declare security headers via the CF zone's Transform Rules instead.

## Troubleshooting

- **Build error: "Dynamic require of '/.next/server/middleware-manifest.json'"** — Run `next build` first, then `npx @opennextjs/cloudflare build --skipNextBuild`. The build:cf script handles this if you ran `npm run build` immediately before.
- **Database connection fails** — Most Vercel apps use `@vercel/postgres`. Switch to `@neondatabase/serverless` and use a Neon pooled URL. The script auto-rewrites if it detects `@vercel/postgres`.
- **Auth.js 302 loops** — If pocket-prompts uses Auth.js with a custom domain via CF Workers Routes, add explicit routes for `/api/auth/*` in wrangler.jsonc (see `apps/harbour/creaseworks/wrangler.jsonc` for the pattern).
