## Recommended guardrails (so this can't recur)

1. **Pre-deploy push gate (highest leverage).** Wrap `wrangler deploy` in an npm script / shell function that refuses to deploy when the tree is dirty or `HEAD` is unpushed:
   ```sh
   # refuse if uncommitted changes under the app dir, or HEAD not on origin
   git diff --quiet && git diff --cached --quiet || { echo "✋ dirty tree — commit first"; exit 1; }
   git merge-base --is-ancestor HEAD "origin/$(git branch --show-current)" 2>/dev/null || { echo "✋ HEAD not pushed"; exit 1; }
   ```
   Add as `"predeploy"` in each app's `package.json` (npm runs it automatically before `deploy`).
2. **Weekly "unpushed deploys" report** — a scheduled job that lists CF Workers whose name has no matching committed `wrangler.*`, plus any local branch with unpushed commits. This audit, automated. (Could run as an Opsy tier check.)
3. **Harden `.gitignore` everywhere** — add a bare `.env` line to root and `port/.gitignore` (and any app using `.env*.local`). A repo-root `.env` catch-all: `.env` and `.env.*` with `!.env.example`.
4. **Pre-commit secret scan** — `gitleaks` (or a tiny pre-commit hook grepping for `CLOUDFLARE_API_TOKEN=…`, `sk-…`, etc.) so a token never enters history locally, even before a push is attempted.
5. **Adopt `git push` from one repo for harbour** — collapse the windedvertigo/harbour-apps duplication to one canonical source to kill the drift ambiguity.
6. **Branch hygiene** — prune harbour-apps' 36 merged branches; push or `save/`-archive windedvertigo's 9 unpushed-work branches.
