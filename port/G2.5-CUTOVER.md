## Step 2 — Confirm Cloudflare DNS is routing correctly

CF Workers routes override DNS for matched patterns. With the route registered,
`port.windedvertigo.com/*` traffic flows to the CF Worker automatically.

Check in CF dashboard or:
```bash
curl -s -o /dev/null -w "%{http_code}" https://port.windedvertigo.com/login
# Expected: 200
```

## Step 3 — Delete Inngest infrastructure

### 3a. Delete the dual-dispatch imports from API routes

In each of these files, remove the `inngest.send()` call and its import:

| File | Line | Action |
|------|------|--------|
| `app/api/rfp-radar/[id]/route.ts` | ~5, ~41 | Remove `import { inngest }` + `inngest.send()` block |
| `app/api/rfp-radar/[id]/document/route.ts` | ~20, ~300, ~316 | Remove `import { inngest }` + 2× `inngest.send()` |
| `app/api/rfp-radar/[id]/document/paste/route.ts` | ~22, ~213 | Remove `import { inngest }` + `inngest.send()` |
| `app/api/rfp-radar/[id]/regenerate-proposal/route.ts` | ~29, ~70 | Remove `import { inngest }` + `inngest.send()` |
| `app/api/timesheets/[id]/route.ts` | ~5, ~38 | Remove `import { inngest }` + `inngest.send()` |

### 3b. Delete Inngest files

```bash
rm -rf port/app/api/inngest/
rm -rf port/lib/inngest/
```

### 3c. Remove `/api/inngest` from middleware allowlist

In `port/middleware.ts`, remove these lines:
```typescript
pathname.startsWith("/api/inngest") ||
// Inngest webhook — ...
// ...
```

### 3d. Remove Inngest package dependency

```bash
cd port && npm uninstall inngest
```

## Rollback plan

If anything goes wrong after DNS cutover:

1. Comment out the `routes` block in `wrangler.jsonc` and redeploy
2. CF route is removed; Vercel `wv-crm.vercel.app` / Vercel domain continues serving
3. Re-add `/api/inngest` to middleware allowlist if Inngest functions need to resume
