Retainer + Buyout Proposal

From Garrett Jaeger · Winded Vertigo  ·  For Nordic Naturals (SQR-RCT + PCS Platform)  ·  Date 2026-04-22

TL;DR

A production regulated-data platform built on LLM-backed pipelines doesn't stand still. Three categories of ongoing work — security maintenance, LLM vendor management, and Nordic-side data evolution — warrant a monthly retainer rather than a project-based engagement. This proposal structures that retainer, defines a clean wind-down or accelerated-buyout clause that protects both parties, and scopes a separate Wave 10 "Evergreen Extraction" deliverable that reduces Anthropic dependency by migrating appropriate workloads to deterministic parsers and open-source LLMs.

Target shape: $6,000/mo Priority-tier retainer, 12-month auto-renewing term, 60-day notice, 6× monthly buyout fee on accelerated exit. Wave 10 scoped at $20–35k, optionally pre-committed in the first 90 days.

1. Why a retainer, not just a project fee

The work to date has delivered a live production platform: PCS documents, canonical claims, label import, SQR-RCT scoring, weekly digests, nightly re-pings, ingredient-safety escalations, in-app feedback fan-out to Slack, bcrypt-authenticated multi-role auth with 1h access tokens and live role re-verification, and an audit trail for all mutations. That platform does not stand still. Three categories of work recur:

  1. Security maintenance — password policy, token rotation, dependency patches, vendor-breach response, regulatory updates (FDA DSHEA, FTC Green Guides, GDPR, CCPA), penetration-test findings, audit-log reviews.
  2. LLM ecosystem churn — Claude, OpenAI, and every downstream LLM provider ship breaking changes on ~quarterly cadence (pricing, rate limits, retention defaults, API shape, model deprecations). Each one is a live risk.
  3. Nordic-side data hygiene — new products, new claim templates, label format revisions, Lauren's template updates, canonical-claim deduplication, study intake changes.

A one-time build fee covers none of these. Project-based engagement creates feast/famine dynamics where Nordic only pays when something is already broken. A retainer aligns incentives: predictable revenue funds proactive maintenance, and Nordic has a committed owner for the platform without renegotiating every quarter.

4. Buyout / termination / transition

The most important section for risk posture: what happens if Nordic decides to stop paying.

Path A — Wind-down (default)

Nordic gives 60-day notice. A 90-day wind-down begins.

  • Retainer continues at current rate through wind-down.
  • Transition package delivered: complete handoff runbook (docs/handoff/*), credentials transfer to Nordic-named successor (Notion, Vercel, Anthropic, Slack, GitHub), live walkthrough sessions (up to 8h), dependency/vendor map + cost attribution, 30-day post-termination email availability (short answers, no billable hours).
  • Open-source migration NOT included in wind-down. If the platform's LLM features matter, Nordic either continues the Anthropic contract themselves or commissions Wave 10 (§5) before terminating.

Path B — Accelerated buyout

Nordic wants immediate exit (no 90-day wind-down).

  • Exit fee = 6 × current monthly retainer, paid on termination.
  • Includes Path A deliverables compressed into 30 days.
  • Protects revenue runway during transition to next engagement.

Termination for cause

If termination is for documented cause (breach, gross negligence, willful misconduct), no buyout fee. Ordinary performance disagreements do NOT constitute cause.

Non-solicitation (recommended optional clause)

If signed, Garrett does not accept engagements from direct Nordic Naturals competitors (US supplement category, top 20 by revenue) for contract term + 12 months post-termination. Addresses Nordic's IP-leak concern without requiring full-time exclusivity.

5. The LLM dependency problem

The risk (frank assessment). The platform's most valuable capabilities — PCS extraction, label OCR, auto-scoring, claim drafting, reformulation suggestions — all depend on Anthropic's Claude API. If that breaks (pricing shock, account suspension, provider bankruptcy, regulatory ban, model deprecation, or Nordic stops paying), affected features fail within hours. This is vendor concentration risk.

It's also why Nordic should retain rather than build-and-abandon: the hardest work is NOT in React code or Notion schema — it's in prompt engineering, confidence gates, extraction validation, and the accumulated corpus of "what Claude gets wrong on Nordic's specific template." That knowledge is not recoverable from a code read-through.

Wave 10 — Evergreen Extraction (separate project)

Reduces LLM dependency through a phased migration:

  1. Deterministic parsers first. Replace LLM calls for well-structured fields with rule-based extraction (SKU/FMT/PCS ID patterns, dose regex, formula-line structure from tables). Free, fast, version-stable, never hallucinate.
  2. Open-source LLMs via managed hosts. Add Llama 3 / Mistral / DeepSeek via Groq, Together.ai, or Replicate as second provider. Adapter pattern — rest of codebase is model-agnostic. Costs drop ~80–95% vs. Claude; models are fungible.
  3. Self-hosted OSS fallback. For most-sensitive content, a self-hosted Llama variant on Vercel Fluid Compute or cheap EC2 GPU. Only warranted if DPO posture requires zero third-party inference.
  4. Vision holdout. Claude vision remains best for label OCR until OSS vision catches up (watch Qwen-VL, InternVL, Pixtral). Keep Claude for labels specifically, OSS for everything else. Clear audit line: "Nordic label images visit Anthropic; nothing else does."
  5. Aggressive caching. Extraction results persisted with content hashes. Re-uploading the same PDF doesn't re-hit any API. Mostly in place; Wave 10 formalizes.

Wave 10 budget: $20,000 – $35,000 one-time. Delivered over 6-10 weeks. Retainer continues throughout — Wave 10 is the retainer's purpose, not its replacement.

If Nordic terminates before Wave 10 ships

This is the scenario the retainer is designed to protect against: "build the hard thing, then get kicked off before the evergreen version lands." Three protections:

  1. The retainer itself. If the LLM features matter to Nordic, they need ongoing maintenance. Terminating without replacement means those features degrade. Nordic's own interest aligns with retention through transition.
  2. Buyout fee (§4). 6× monthly retainer on accelerated exit recoups some of the platform-specific IP investment.
  3. Wave 10 pre-commitment. Strong recommendation: Wave 10 scoping conversation happens in the first 90 days of the retainer. If Nordic commits to Wave 10 funding up front, both parties have aligned incentive to see it through.

8. Why Nordic should sign this

  1. Predictable monthly cost vs. panic-fee emergencies when something breaks.
  2. Named technical owner — research team knows who to call and their response-time commitment.
  3. Security maturity baseline — quarterly reviews + ongoing patching is what auditors want to see.
  4. Vendor-risk management — Nordic doesn't have to become expert in Anthropic's changelogs or Notion's upgrade paths.
  5. Optionality on LLM migration — Wave 10 available at a known price, known outcomes.
  6. Protection against institutional-knowledge loss — platform carries tacit knowledge not recoverable from a code read-through.
