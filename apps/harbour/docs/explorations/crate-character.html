crate sketch — v1

  • Archetype: the builder-organiser. Validates "it's okay to like order and stack things."
  • Form: rigid box. Hosts DB form_primary values like module, construction, carton, block, brick, box, and thick cardboard.
  • Motion vocabulary: deliberate, step-by-step, satisfying thunks — unlike Cord or Swatch, Crate's default state is STILL. The .thunk animation fires briefly every 6.5s (a tiny vertical bounce, like something just landed inside), then returns to rest. .crateblink is a slow deliberate blink every 5.5s. No sway. The motion mismatch is intentional — Crate's confidence comes from stillness.
  • Tell: a visible dent in the top-left corner, compressed inward. Distinguishes Crate at silhouette level from a perfectly-rectangular box.
  • Secondary details that appear on all poses: vertical tape seam down the middle (kraft-lt strip with ink border lines), faint vertical corrugation lines, small redwood "up" stamp with arrow.
  • Eyes sit on the front face, spaced slightly wider than other characters — conveying confidence and readiness.
  • Function poses: contain (lid open with three objects peeking — ball, block, star), stack (smaller crate on top showing load-bearing), fort (arched doorway cut in front, darker interior, flag on top-right corner).
  • The fort pose uses fill-rule="evenodd" on the body path to carve the doorway cleanly — same technique Jugs uses for the handle hole.
